# EULA 同意機能 要件定義書

> Issue: #81
> 生成日時: 2026-03-04
> ステータス: Draft

## 1. 概要

App Store Review Guideline 1.2 (Safety - User-Generated Content) への対応として、
ユーザーに利用規約（EULA）への同意を求める機能を実装する。

## 2. 背景・動機

- App Store Review で Guideline 1.2 に抵触しリジェクトされた
- UGC（ユーザー生成コンテンツ）を含むアプリには EULA 同意が必須
- 不適切なコンテンツ・悪質なユーザーに対するゼロトレランスポリシーの明記が必要

## 3. 機能要件

### FR-1: EULA 同意画面の表示

- アプリ起動時、EULA に未同意の場合は同意画面を表示する
- 同意するまでアプリの主要機能（フィード・投稿・マイ短歌）にアクセスできない
- 匿名認証完了後、ContentView 表示前に同意チェックを行う

### FR-2: EULA 文面

利用規約には以下を含む:
- 不適切なコンテンツ（暴力的・差別的・性的・違法な内容）の投稿禁止
- 悪質なユーザーに対するゼロトレランスポリシー（アカウント停止・削除の可能性）
- 通報機能の存在と運営による対応
- 個人情報を含む投稿の禁止
- コンテンツが他ユーザーに公開される旨

### FR-3: 同意操作

- 利用規約全文をスクロールで閲覧できる
- 「同意する」ボタンで同意を確定する
- 同意しない場合はアプリを利用できない（同意画面のまま）

### FR-4: 同意状態の永続化

- 同意状態を UserDefaults に保存する（`@AppStorage`）
- 一度同意すれば再表示しない
- アカウント削除時は同意状態もリセットする

### FR-5: 設定画面からの利用規約閲覧

- 設定画面の「情報」セクションに「利用規約」リンクを追加する
- タップすると EULA 全文を閲覧できる（同意ボタンなし、閲覧のみ）

## 4. 非機能要件

- EULA 文面はアプリ内に静的に埋め込む（サーバー取得不要）
- 日本語のみ対応
- iOS 17+ 対応

## 5. 受け入れ条件

- [ ] 未同意状態でアプリを起動すると EULA 同意画面が表示される
- [ ] EULA に Guideline 1.2 の要件（ゼロトレランスポリシー）が含まれている
- [ ] 「同意する」ボタンで同意すると主要機能にアクセスできる
- [ ] 同意状態が永続化され、再起動後は同意画面が表示されない
- [ ] アカウント削除後に再認証すると、再度同意画面が表示される
- [ ] 設定画面から利用規約を閲覧できる
